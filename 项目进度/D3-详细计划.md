## 二、数据模型详细设计

### 2.1 Video（视频）— 核心实体，~100 条

```python
class Video(BaseModel):
    video_id: str          # 格式: v_movie_001, v_tv_001, v_variety_001 ...
    title: str             # 视频标题
    original_title: str | None  # 原名/英文名（可选）
    description: str       # 剧情简介（50-200字）
    year: int              # 上映年份（2000-2026）
    region: str            # 地区：中国大陆 / 中国香港 / 美国 / 韩国 / 日本 / 英国
    language: str          # 语言：国语 / 英语 / 韩语 / 日语
    type: VideoType        # 枚举: movie / tv / variety / animation / documentary
    genres: list[str]      # 风格标签: 喜剧 / 动作 / 科幻 / 爱情 / 悬疑 / 剧情 / 古装 / 奇幻 / 犯罪 / 恐怖 / 战争 / 动画 / 纪录片 / 真人秀 / 脱口秀
    rating: float          # 评分 1.0-10.0（腾讯视频风格，保留1位小数）
    vote_count: int        # 评分人数（1000-500000）
    episode_count: int | None  # 集数（电视剧/动漫/综艺），电影为 None
    duration: int | None       # 时长分钟（电影），剧集为 None
    status: str            # completed / ongoing / upcoming
    poster_url: str        # 占位海报 URL
    director_ids: list[str]    # 关联导演 ID 列表 [d_001, d_002]
    actor_ids: list[str]       # 关联主演 ID 列表 [a_001, a_002, a_003]
```

## 三、数据分布策略

### 3.1 类型分布（~100 条）

| 类型 | 数量 | 年代分布 | 评分区间 |
|------|------|----------|----------|
| 电影（movie） | 40 条 | 2000-2026 | 5.5-9.5 |
| 电视剧（tv） | 25 条 | 2005-2026 | 6.0-9.3 |
| 综艺（variety） | 15 条 | 2010-2026 | 6.5-9.0 |
| 动漫（animation） | 12 条 | 2000-2026 | 6.0-9.8 |
| 纪录片（documentary） | 8 条 | 2005-2026 | 7.0-9.6 |

### 3.2 地区分布

- 中国大陆：~60%
- 美国：~15%
- 韩国：~10%
- 日本：~10%
- 其他（英国/中国香港等）：~5%

### 3.3 关联关系

- 每个 Video 关联 1-2 位 Director
- 每个 Video 关联 2-5 位 Actor
- 每个 Video 关联 1-3 个 Category
- 每个 Actor 出演 2-8 部作品
- 每个 Director 执导 2-6 部作品

## 四、脚本设计 (`generate_data.py`)

### 4.1 脚本结构

```
generate_data.py
├── 1. 配置区
│   ├── SEED = 42（可复现性）
│   ├── 预设演员/导演/分类元数据（硬编码名称池）
│   └── 各类型条目数量配置
│
├── 2. 生成器函数
│   ├── generate_categories()    → list[Category]
│   ├── generate_actors(n)       → list[Actor]
│   ├── generate_directors(n)    → list[Director]
│   ├── generate_videos(n, types_distribution, actors, directors, categories)
│   │                              → list[Video]
│   └── generate_all()           → 总入口
│
├── 3. 导出函数
│   ├── export_to_json(data, filepath)
│   └── main()                   → 一键生成全部 JSON
│
└── 4. main 入口
    └── if __name__ == "__main__": main()
```

### 4.2 关键设计原则

- **确定性**：使用 `random.seed(42)`，每次运行生成完全相同的数据
- **贴近真实**：标题/简介参考腾讯视频真实片库风格（如《庆余年》《三体》《斗罗大陆》等风格）
- **Pydantic 校验**：所有实体用 Pydantic BaseModel，生成时自动校验字段合法性
- **关联完整性**：确保所有引用的 actor_id / director_id / category_id 真实存在

### 4.3 运行方式

```bash
cd tencent_video_agent
python data/generate_data.py
```

输出文件：
```
data/
├── categories.json      # ~15 条
├── actors.json          # ~60 条
├── directors.json       # ~30 条
└── videos.json          # ~100 条
```
