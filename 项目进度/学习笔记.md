### 1. 学习笔记：LangGraph 核心概念（结构化总结）

#### **1. LangGraph 是什么？**
LangGraph 是 LangChain 生态中用于构建**可靠、可控、可视化、多 Agent 状态机工作流**的库。它将 LLM 应用建模为**有向图（State Machine）**，完美解决传统 Chain 在复杂场景下的“不可控、不可持久、无循环”问题。

**核心优势**（对比 LangChain Chain）：
- **状态持久化**：内置 Checkpoint（内存/SQLite/Postgres）
- **循环与条件分支**：支持条件边（Conditional Edges）
- **多 Agent 协作**：每个 Agent 就是一个 Node
- **可视化 & 调试**：原生支持 Mermaid + LangSmith
- **生产就绪**：Human-in-the-loop、Streaming、Persistence

#### **2. 核心概念详解**

**（1）State（状态）**
- 整个工作流共享的**结构化数据容器**（推荐使用 `TypedDict` + `Annotated`）。
- 支持 **Reducer**（如何合并多个 Node 的更新，例如 `add_messages`）。
- 示例：
```python
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph.message import add_messages

class AgentState(TypedDict):
    messages: Annotated[list, add_messages]  # 对话历史（自动追加）
    user_intent: str
    retrieved_docs: list
    plan: dict
    next: str  # 路由控制
```

**（2）Nodes（节点）**
- 任意 **callable**（函数、async 函数、Runnable）。
- 输入：当前 State；输出：**部分 State 更新**（dict）。
- 典型 Node 类型：
    - LLM Agent Node（带 Tools）
    - Tool Node
    - Router Node（意图识别）
    - Human Feedback Node

**（3）Edges（边）**
- **普通边**：`add_edge(source, target)`
- **条件边**：`add_conditional_edges(source, condition_func, mapping)`
- **特殊节点**：`START`、`END`

**（4）StateGraph & 编译**
```python
from langgraph.graph import StateGraph, START, END

graph_builder = StateGraph(AgentState)
graph_builder.add_node("intent", intent_agent)
# ... 添加更多节点和边
graph = graph_builder.compile(checkpointer=memory)
```

**（5）Memory（记忆）**
- `MemorySaver()`（内存）
- `SQLiteSaver` / `PostgresSaver`（持久化）
- 支持线程级（thread_id）多会话隔离

### 2. 技术调研报告：LangGraph 在多 Agent 系统中的应用

**报告日期**：2026-04-29  
**作者**：专业大模型Agent应用开发助手

#### **调研结论**
LangGraph 是目前构建**生产级多 Agent 系统**的最优框架，尤其适合本项目“腾讯视频智能观影助手”这种**意图分支 + 多工具 + RAG + 知识图谱 + 推荐生成**的复杂场景。

**关键适用性**：
1. **意图识别 → 分支路由**：条件边完美实现“找片/咨询/计划/闲聊”四路分流。
2. **Agent 协作**：每个 Agent（意图、检索、知识查询、推荐、对话管理）作为一个 Node，State 在它们之间自然流转。
3. **RAG + 工具调用**：Node 内可无缝集成 Chroma、Neo4j、自定义 Tools。
4. **用户历史 & 个性化**：Memory + SQLite 用户画像持久化。
5. **调试 & 可观测**：LangSmith 一键追踪每条边、每个 Node 的输入输出。

**与其它方案对比**：
| 方案 | 状态管理 | 条件分支 | 可视化 | 生产特性 | 推荐指数 |
|------|----------|----------|--------|----------|----------|
| LangChain Chain | 弱 | 差 | 无 | 一般 | ★★ |
| CrewAI/AutoGen | 中 | 中 | 一般 | 中 | ★★★ |
| **LangGraph** | 强 | 极强 | 原生Mermaid | 优秀 | **★★★★★** |

**潜在风险 & 应对**：
- State 设计过大 → 使用 Pydantic + 精简字段 + Reducer
- 死循环 → 设置 `recursion_limit` + 终止条件
- 性能 → 使用 async + 批量检索

由于无法直接手绘，我使用 **Mermaid**（LangGraph 官方推荐可视化方式）给出清晰流程图。您可复制到 [mermaid.live](https://mermaid.live) 查看/导出图片。

```mermaid
%%{init: {'flowchart': {'curve': 'linear'}}}%%
flowchart TD
    subgraph 用户输入
        START[用户消息] 
    end

    START --> INTENT[意图识别 Agent\n(多轮对话分类)]
    
    INTENT -->|找片| RETRIEVAL[视频检索 Agent\n(Chroma + 过滤)]
    INTENT -->|咨询| KG[知识查询 Agent\n(Neo4j + RAG)]
    INTENT -->|计划| PLAN[观影计划 Agent]
    INTENT -->|闲聊| CHAT[对话管理 Agent]
    
    RETRIEVAL --> RECOMMEND[推荐生成 Agent]
    KG --> RECOMMEND
    PLAN --> RECOMMEND
    
    RECOMMEND --> RESPONSE[最终回复生成 + 记忆更新]
    RESPONSE --> END[结束 / 返回前端]
    
    %% 循环与反馈
    RESPONSE -.->|需要澄清| INTENT
    RECOMMEND -.->|Human Feedback| HUMAN[人工干预节点]
```

**流程说明**：
- **橙色**：条件分支（由意图 Agent 决定）
- **蓝色**：核心业务 Agent
- **虚线**：循环/反馈路径（多轮对话）
