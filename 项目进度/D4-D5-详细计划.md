## 四、数据入库（SQLite + Chroma）

### 4.1 SQLite 数据导入

**设计思路**：将 JSON 数据导入 SQLite 关系表，作为业务数据的主存储。

**表结构设计**：

```sql
CREATE TABLE videos (
    video_id TEXT PRIMARY KEY,
    title TEXT NOT NULL,
    original_title TEXT,
    description TEXT,
    year INTEGER,
    region TEXT,
    language TEXT,
    type TEXT,
    rating REAL,
    vote_count INTEGER,
    episode_count INTEGER,
    duration INTEGER,
    status TEXT
);

CREATE TABLE actors (
    actor_id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    gender TEXT,
    birth_year INTEGER,
    nationality TEXT,
    bio TEXT,
    popularity INTEGER
);

CREATE TABLE directors (
    director_id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    birth_year INTEGER,
    nationality TEXT,
    bio TEXT,
    popularity INTEGER
);

CREATE TABLE categories (
    category_id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    description TEXT
);

-- 关联表
CREATE TABLE video_actors (
    video_id TEXT,
    actor_id TEXT,
    PRIMARY KEY (video_id, actor_id),
    FOREIGN KEY (video_id) REFERENCES videos(video_id),
    FOREIGN KEY (actor_id) REFERENCES actors(actor_id)
);

CREATE TABLE video_directors (
    video_id TEXT,
    director_id TEXT,
    PRIMARY KEY (video_id, director_id),
    FOREIGN KEY (video_id) REFERENCES videos(video_id),
    FOREIGN KEY (director_id) REFERENCES directors(director_id)
);
```

### 4.2 Chroma 向量数据库导入

**设计思路**：将视频的文本信息（标题+简介+类型）向量化存入 Chroma，用于后续语义检索。

- **Collection 名**：`tencent_videos`
- **Embedding 模型**：`sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2`（支持中文）
- **每条 Document 内容**：`{title}\n{description}\n{genres}`
- **Metadata**：video_id, title, year, type, rating, region
- **Chunk 策略**：每条视频作为一个 Document 条目
